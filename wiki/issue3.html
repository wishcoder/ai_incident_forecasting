Incident Details

Incident ID:003

Title:Router-1 Configuration Changes

Date and Time:2023-04-05 14:20

Reported By:Cynthia Moore/IT Support

Component Affected:Router-1

Incident Description

Unscheduled configuration changes to Router-1 led to a loss of external connectivity for App-1 and App-2. The changes were made inadvertently during routine maintenance.

Impact Analysis

The configuration changes disrupted normal operations, preventing external access to App-1 and App-2. This incident affected user transactions and data processing capabilities.

Resolution Steps

1. The issue was identified as a misconfiguration during a routine review.

2. Configuration changes were rolled back to the previous stable state at 15:00.

3. Full connectivity for App-1 and App-2 was restored by 15:30.

Resolved By:David Lee/Network Engineering

Resolution Date and Time:2023-04-05 15:30

Status

Resolved

Lessons Learned

The incident highlighted the need for stricter change management protocols and pre-approval processes for any configuration changes to critical network components. Additionally, it underscored the importance of immediate rollback capabilities and the need for comprehensive logging of all changes.
